# 📁 CodeHub - بستری مدرن برای مدیریت و اشتراک‌گذاری مخازن کد

**CodeHub** یک اپلیکیشن وب تمام‌قدَر (Full-Stack) با فرانت‌اند React است که به کاربران اجازه می‌دهد مخازن کد خود را ایجاد، مدیریت، آپلود فایل، مشاهده درختی فایل‌ها، نمایش فایل‌های متنی با هایلایت syntax، و مطالعه فایل‌های README با پشتیبانی از راست‌به‌چپ (RTL) و چپ‌به‌راست (LTR) انجام دهند. این پروژه با الهام از گیت‌هاب اما با رویکردی ساده‌تر و رابط کاربری شیشه‌ای (Glassmorphism) طراحی شده است.

## ✨ ویژگی‌های اصلی CodeHub

- **🔐 احراز هویت کامل**  
  ثبت‌نام، ورود، نمایش اطلاعات کاربر و خروج امن با ذخیره توکن در `localStorage`

- **📚 مدیریت مخازن**  
  ایجاد مخزن جدید (عمومی/خصوصی)، ویرایش توضیحات و زبان برنامه‌نویسی، حذف مخزن

- **📂 ساختار درختی فایل‌ها**  
  نمایش فایل‌ها و پوشه‌ها به صورت درخت (`Tree View`) با قابلیت بستن/باز کردن پوشه‌ها

- **📄 نمایشگر هوشمند فایل**  
  - هایلایت syntax با `react-syntax-highlighter` (تم Atom Dark)
  - شناسایی خودکار زبان از روی پسوند فایل
  - پشتیبانی از بیش از ۳۰ نوع فایل با آیکون اختصاصی
  - دکمه کپی کد و دانلود فایل

- **📖 نمایش README.md**  
  استفاده از `react-markdown` + `remark-gfm` (جدول، لینک، تصویر) + `rehype-raw` (HTML مستقیم)  
  **پشتیبانی از دو جهت متن** (RTL برای فارسی/عربی و LTR برای انگلیسی) با کلید تعویض لحظه‌ای  
  **لنکرهای داخلی** (`#heading`) با اسکرول خودکار و در نظر گرفتن ارتفاع هدر

- **📎 آپلود فایل به سه روش**  
  1. آپلود تکی/چندتایی فایل (با امکان تعیین مسیر هدف)  
  2. آپلود کل پوشه (حفظ ساختار)  
  3. آپلود فایل ZIP (استخراج خودکار سمت سرور)

- **⬇️ دانلود**  
  دانلود فایل جداگانه یا کل مخزن به صورت ZIP

- **🎨 تجربه کاربری جذاب**  
  - مکان‌نمای سفارشی (`Custom Cursor`) با افکت هاله‌ای  
  - پس‌زمینه گرادیان متحرک و افکت شیشه مات  
  - انیمیشن‌های نرم (Fade-in، Skeleton Loader)  
  - تم دارک مدرن با رنگ‌های نئونی بنفش  
  - واکنش‌گرا (Responsive) برای موبایل، تبلت و دسکتاپ

- **⚡ کارایی بالا**  
  ساخته شده با **Vite** برای بیلد سریع و HMR فوق‌العاده

## 🚀 نصب و راه‌اندازی (Local Development)

1. **کلون مخزن**
   ```bash
   git clone https://github.com/your-username/codehub.git
   cd codehub
   ```

2. **نصب وابستگی‌ها**
   ```bash
   npm install
   # یا
   yarn install
   ```

3. **تنظیم آدرس API**  
   فایل `src/services/api.js` را باز کرده و در خط `API_BASE_URL` آدرس سرور خود را قرار دهید:
   ```js
   const API_BASE_URL = 'https://your-backend-url.com/api/v1';
   ```

4. **اجرا در حالت توسعه**
   ```bash
   npm run dev
   ```
   اپلیکیشن روی `http://localhost:5173` در دسترس خواهد بود.

5. **ساخت نسخه تولید (Production)**
   ```bash
   npm run build
   ```
   خروجی در پوشه `dist/` قرار می‌گیرد.  
   برای پیش‌نمایش بیلد:
   ```bash
   npm run preview
   ```

    ├── components/
    │   ├── common/          # کامپوننت‌های اشتراکی
    │   │   ├── Navbar.jsx       # نوار ناوبری با دکمه New، جستجو، منوی کاربر
    │   │   ├── Footer.jsx

## 📚 راهنما و استفاده

### 1. ساخت حساب کاربری
- در صفحه اصلی، روی دکمه **New** یا آواتار (بالا راست) کلیک کنید.
- گزینه **Sign Up** را انتخاب کنید.
- نام کاربری، ایمیل و رمز عبور (با نمایشگر قدرت رمز) را وارد کنید.
- پس از ثبت‌نام، به طور خودکار وارد می‌شوید.

### 2. ایجاد مخزن جدید
- از داشبورد (**Dashboard**) روی دکمه **Create Repository** کلیک کنید.
- نام مخزن (فقط حروف، اعداد، خط تیره و زیرخط)، زبان برنامه‌نویسی، توضیحات و وضعیت visibility (عمومی/خصوصی) را مشخص کنید.

### 3. آپلود فایل
- در داشبورد، کارت مخزن مورد نظر را پیدا کنید و روی آیکون **Upload** (ابر با فلش به بالا) کلیک کنید.
- بین سه حالت آپلود یکی را انتخاب کنید:
  - **Single/Multiple Files** : فایل‌های دلخواه (می‌توانید یک مسیر هدف مثل `src/components/` وارد کنید)
  - **Entire Folder** : کل یک پوشه از سیستم خود را با حفظ ساختار آپلود کنید
  - **Zip Extract** : یک فایل ZIP را آپلود کنید تا سرور خودکار آن را استخراج کند
- فایل‌ها را انتخاب کرده و دکمه **Start Upload Process** را بزنید.

### 4. مشاهده مخزن و فایل‌ها
- روی هر کارت مخزن (در صفحه اصلی یا داشبورد) کلیک کنید.
- در صفحه مخزن:
  - **سمت چپ**: درخت پوشه‌ها و فایل‌ها (با آیکون‌های رنگی)
  - **سمت راست**: 
    - اگر یک فایل متنی را کلیک کنید، با هایلایت syntax نمایش داده می‌شود (دکمه کپی و دانلود نیز موجود است).
    - اگر `README.md` در ریشه باشد، به صورت خودکار با قابلیت تغییر جهت (RTL/LTR) و لینک‌های داخلی نمایش داده می‌شود.
- دکمه **دانلود ریپازیتوری** در بالای صفحه، کل مخزن را به صورت ZIP دریافت می‌کند.

### 5. ویرایش و حذف مخزن
- به **Dashboard** بروید.
- روی آیکون **مداد (Edit)** در کارت مخزن کلیک کنید.
- تغییرات را اعمال کنید یا از آیکون **سطل زباله (Delete)** برای حذف استفاده کنید.

## 💡 نکات فنی مهم

### 🔮 مکان‌نمای سفارشی
در دستگاه‌های دارای ماوس (`pointer: fine`) نشانگر پیش‌فرض مخفی شده و دو عنصر `ch-cursor-core` و `ch-cursor-halo` جایگزین می‌شوند. با هاور روی لینک‌ها و دکمه‌ها، هاله بزرگتر می‌شود.

### 🎨 تم و افکت‌ها
تم پروژه بر پایه متغیرهای CSS در `:root` است و به راحتی قابل شخصی‌سازی می‌باشد. افکت شیشه‌ای از طریق `backdrop-filter: blur()` اعمال شده است.

### 🌐 پشتیبانی از RTL در README
کامپوننت `ReactMarkdown` در `RepoDetail.jsx` با دو کلید مجزا به کاربر اجازه می‌دهد جهت کل صفحه README را تغییر دهد. همچنین لینک‌های هشداردهنده (مثل `[!NOTE]`) به صورت کارت‌های رنگی زیبا نمایش داده می‌شوند.

### 📦 آپلود فایل‌ها
- برای آپلود پوشه از ویژگی `webkitdirectory` در input فایل استفاده شده است.
- برای آپلود ZIP، هدر `Content-Type: multipart/form-data` تنظیم می‌شود.
- تمام درخواست‌های آپلود به سرور از طریق `fileService.uploadFiles` یا `fileService.importZip` ارسال می‌شوند.

### 🔒 امنیت
توکن JWT در `localStorage` ذخیره و به صورت خودکار در تمام درخواست‌های Axios به هدر `Authorization: Bearer <token>` اضافه می‌شود. در صورت دریافت 401، کاربر به صفحه لاگین هدایت می‌شود.

## 🔌 API Reference (خلاصه)

تمام فراخوانی‌ها در `src/services/api.js` متمرکز شده‌اند:

| سرویس | متد | endpoint | توضیح |
|-------|------|----------|-------|
| `authService.login` | POST | `/auth/login` | ورود با username و password |
| `authService.register` | POST | `/auth/register` | ثبت‌نام کاربر جدید |
| `repoService.getAll` | GET | `/repo/getAll` | دریافت تمام مخازن عمومی |
| `repoService.getUser` | GET | `/repo` | دریافت مخازن کاربر جاری |
| `repoService.create` | POST | `/repo` | ایجاد مخزن جدید |
| `repoService.update` | PUT | `/repo` | بروزرسانی مخزن |
| `repoService.delete` | DELETE | `/repo` | حذف مخزن (شناسه در body) |
| `fileService.getFiles` | GET | `/file/{repoId}/files` | دریافت لیست فایل‌های مخزن |
| `fileService.getContent` | GET | `/file/{repoId}/files/{fileId}/content` | دریافت محتوای فایل متنی |
| `fileService.downloadFile` | GET (blob) | `/file/{repoId}/files/{fileId}/download` | دانلود یک فایل |
| `fileService.downloadRepository` | GET (blob) | `/file/{repoId}/download` | دانلود کل مخزن به صورت ZIP |
| `fileService.uploadFiles` | POST (form-data) | `/file/{repoId}/files` | آپلود فایل(ها) |
| `fileService.importZip` | POST (form-data) | `/file/{repoId}/import` | آپلود و استخراج ZIP |

> توجه: سرور باید این endpoints را مطابق مستندات پیاده‌سازی کرده باشد.
